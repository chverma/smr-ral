Resumen

La telemática es la ciencia que estudia las técnicas necesarias para la transmisión de datos, entre un emisor y un receptor, por medio de una red de telecomunicaciones.

Definimos la transmisión de datos como la transferencia de información codificada de un punto a otro, u otros, mediante señales eléctricas, electroópticos o electromagnéticos.

La comunicación es el proceso mediante el cual se transporta información; esta información viaja en una señal que se transmite desde el emisor hasta el receptor. El emisor y el receptor conocen el mismo código; por eso son capaces de interpretar las señales para obtener la información.

Un módem -modulador y desmodulador- convierte la señal digital en analógico y, cuando llega al receptor, transforma la señal analógica en digital.

Los estándares de redes proporcionan la base para la transmisión de datos, para la fabricación de equipos de red compatibles y para el diseño de sistemas operativos de red.

Hay dos modelos de comunicación principales que utilizan capas: el modelo de referencia OSI (open system interconnection, interconexión de sistemas abiertos ') y el modelo de protocolo TCP / IP (transporte Control Protocol / Internet Protocol,' protocolo de control de transporte / protocolo de Internet ') o modelo de Internet.

Los protocolos de comunicación son conjuntos de normas y procedimientos empleados para realizar una comunicación. Controlan todos los aspectos de la comunicación de datos:

  • Cómo se construye la red física.
  • Elemento de lista de picos.
  • Como se conectan los ordenadores en la red.
  • Como se da formato a los datos para transmitirlas.
  • Como se controlan los errores.
  • Etc.

El modelo de referencia OSI, lanzado en 1984, es el esquema descriptivo que creó la ISO. Este modelo proporciona a los fabricantes un conjunto de normas que facilitan una compatibilidad y una operabilidad más grandes entre los tipos de tecnologías de red producidos por varias empresas de todo el mundo.

Para cada una de las capas del modelo de referencia OSI hay definidas un conjunto de normas o características que debe cumplir; es decir, cada capa dispone de un protocolo específico para suministrar los servicios que le son encomendados.

La capa de aplicación es la capa del modelo OSI más cercana al usuario, ya que provee de servicios de red sus aplicaciones. Es la única capa que no proporciona servicios a ninguna otra capa OSI, sino sólo a aplicaciones que se encuentran fuera del modelo OSI.

Las misiones principales de la capa de sesión son:

  • Iniciar, administrar y terminar las sesiones entre dos ordenadores que se comunican.
  • Proporcionar sus servicios a la capa de presentación.
  • Sincronizar el diálogo entre las capas de presentación de los dos ordenadores.
  • Regular la sesión.

La capa de transporte segmenta los datos originales en el ordenador emisor y las asemeja dentro del sistema del ordenador receptor.

La capa de red proporciona conectividad y selección de ruta entre dos sistemas de computadoras que pueden estar ubicados en redes geográficamente distintas.

La capa de enlace de datos proporciona tráfico de datos de confianza por medio de un enlace físico. Esta capa se ocupa del direccionamiento físico, de la topología de red, del acceso a la red, de la notificación de errores, de la entrega ordenado de tramas y del control de flujo.

La capa física define las especificaciones eléctricas, mecánicas, de procedimiento y funcionales para activar, mantener y desactivar el enlace físico entre sistemas finales.

La capa de aplicación TCP / IP se corresponde con las capas de aplicación, presentación y sesión del modelo OSI.

La capa de transporte TCP / IP se corresponde con la capa de transporte del modelo OSI.

TCP es un protocolo fiable orientado a la conexión, que hace que un flujo de bytes o bytes de la aplicación de la máquina origen sea entregado sin errores en la aplicación de cualquier máquina destino de red.

UDP es un protocolo no fiable, sin conexión, para aplicaciones que no requieren ni la asignación de una secuencia ni el control de flujo de TCP, o que quieren utilizar sus medios de control propios.

La capa de Internet del modelo TCP / IP es equivalente a la capa de red del modelo OSI.

La capa de acceso a la red del modelo TCP / IP es equivalente a las capas de enlace y física del modelo OSI.

El propósito de la capa física es transportar una corriente de bits en bruto de una máquina a otra.

Los dispositivos de red se conectan a otros dispositivos, principalmente ordenadores centrales. Estos dispositivos mueven o controlan el tráfico de la red. Los concentradores (hubs), los conmutadores (switches) y los enrutadores (routers) son algunos ejemplos de dispositivos de red.

Hay varias clasificaciones de redes, según el criterio que se emplee:

  • La extensión geográfica
  • La titularidad de la red
  • Cliente-servidor (de igual a igual, peer-to-peer)
  • La topología de la red

Las LAN están diseñadas para hacer lo siguiente:

  • Operar en un área geográfica limitada (un edificio).
  • Permitir a los usuarios acceder a servicios de banda ancha.
  • Proporcionar conectividad con los servicios locales.
  • Conectar dispositivos adyacentes.

Las MAN están diseñadas para proporcionar a los usuarios la distribución de datos interconectando las diversas XAL.

Las WAN están diseñadas para dar conexión a equipos en un área geográfica muy extensa.

La topología física nos dice como es la distribución del cableado. La topología lógica nos dice cómo acceden al medio los datos.
